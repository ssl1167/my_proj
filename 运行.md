
假设你的数据根目录是 `data/data_1`，且你希望将实验结果保存在 `outputs/final_experiment` 中。

### 1. 第一阶段：数据集预处理与拆解

首先，确保你的数据已经准备好。如果你的数据是原始的 `qasm` 文件，需要先将其通过 `build_data1_dataset.py` 转化为标准格式。

```bash
# 运行拆解脚本，将 data_1 下的线路转化为 train/valid/test 集合
python build_data1_dataset.py --dataset_dir data/data_1 --num_circuits 200

```

### 2. 第二阶段：核心强化学习训练（含路由后端与基线锁定）

这是整个流程的核心，包含了之前所有优化的闭环逻辑。我们将训练参数设置为标准的 **"sabre"** 基线锚定和 **"qiskit"** 路由，这会触发我们精心设计的缓存机制，大幅加速训练。

```bash
# 执行训练脚本，建议在 CUDA 设备上运行
python train.py \
    --dataset_dir data/data_1 \
    --save_dir outputs/final_experiment \
    --topology_mode heavy_hex \
    --topology_distance 5 \
    --baseline_mode sabre \
    --router_backend qiskit \
    --action_mode hierarchical \
    --curriculum 12 16 20 24 28 32 36 \
    --epochs_per_stage 10 \
    --episodes_per_epoch 50 \
    --eval_episodes 20 \
    --seed 7

```

### 3. 第三阶段：最终策略性能评估

训练完成后，使用 `evaluate.py` 对模型在测试集上进行最终审计，确保没有发生过拟合。

```bash
# 使用训练出的 best_model 进行全面性能 audit
python evaluate.py \
    --dataset_dir data/data_1 \
    --model_path outputs/final_experiment/best_model.pt \
    --eval_manifest data/data_1/test.json \
    --router_backend qiskit

```

### 💡 给学者专家的额外 Check-list (启动前必读)

在正式按下回车前，请确认这 3 个隐藏细节，这将决定你后续生成图表的真实性：

1. **环境依赖检测**：
确保运行环境下已安装 `pytket` 和 `pytket-qiskit`。如果你的训练任务较重，建议在 `requirements.txt` 基础上额外执行一次：
`pip install pytket pytket-qiskit`
这样即便你在配置中切换到 `router_backend="tket"`，代码也能流畅运行。
2. **验证数据完整性**：
在执行 `train.py` 之前，运行一次 `check_circuit_sizes.py`，确保 `data_1` 里的电路没有因为文件读取错误或编码问题而变成空线路，否则 `env.reset` 会因为逻辑门数量为 0 而抛出异常。
3. **关于 `terminal_scale` 的调优策略**：
如果你在训练日志 `history.jsonl` 中发现 `train_reward` 震荡极度剧烈，请立即将 `terminal_scale` 从 `1.0` 调整为 `0.5` 或 `2.0`。这是你目前唯一需要根据芯片拓扑复杂程度进行微调的“超参数天平”。
